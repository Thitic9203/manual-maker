# รีวิว 5 ด่าน (การ์ด) + Delivery Gate — confluence-docs

**บังคับ — ห้ามข้าม ห้ามลัด.** หน้า Confluence ที่ draft แล้ว **ยังไม่ใช่ของที่ publish/ส่งได้**. ต้องผ่าน
**ครบทั้ง 5 ด่าน** โดยแต่ละด่านตัดสินด้วย **หลักฐานที่จับต้องได้** ไม่ใช่ความรู้สึก. โครงยึดตาม
`manual-maker/references/review.md` ปรับให้เข้ากับ Confluence และทำเป็น **การ์ดรายด่าน** เพื่อไล่ทีละข้อไม่ให้หลุด.

---

## กฎเหล็กของ Gate (อ่านก่อน — ทุกด่านอยู่ใต้กฎนี้)

> **ห้ามบอกว่า "เสร็จ / publish ได้ / เรียบร้อย" จนกว่าทั้ง 5 ด่านจะขึ้น ✅ ครบ 5/5.**

1. **ห้ามมโน (ในเนื้อหา).** ทุกค่าที่ใส่แทน placeholder ต้องสาวกลับไปถึงแหล่งจริงได้. ค่าที่แหล่งไม่ได้บอก →
   **หยุด ถามผู้ใช้** ห้ามเดา ห้ามประมาณ ห้ามเติมให้ครบเอง.
2. **ห้ามโกหก (ในผลรีวิว).** ห้ามขึ้น ✅ ให้ด่าน/ข้อที่ **ยังไม่ได้ตรวจจริง**. ตรวจไม่ได้ (ยังไม่ publish,
   ไม่มีต้นฉบับมาเทียบ, ไม่มีแหล่งมายืนยัน, ยังไม่ได้ screenshot) → ข้อนั้น **❌ ตรวจไม่ได้ = ไม่ผ่าน** แล้วบอก
   ตรงๆ ว่าตรวจอะไรไม่ได้และทำไม. **"น่าจะผ่าน / น่าจะโอเค / คิดว่าถูก" = โกหก ห้ามใช้.**
3. **เจอ ❌ ข้อเดียว = แก้ แล้วรีวิว _ใหม่ทั้ง 5 ด่าน_.** ไม่ใช่รีวิวเฉพาะข้อที่แก้ — แก้ด่านหนึ่งพังอีกด่านได้
   (เพิ่มแถว → เลขข้อเลื่อน; แก้ศัพท์ → index ไม่ตรง child; ใส่ diagram → โครงเปลี่ยน).
4. **ลำดับบังคับ:** ด่าน 1–4 พิสูจน์บน **body ที่เตรียมไว้ _ก่อนเขียน_** · ด่าน 5 พิสูจน์บน **หน้าที่ publish จริง**.
   การ publish เป็น outward-facing ไปยัง space ที่ทีมใช้ — **ห้ามเกิดก่อนด่าน 1–4 ผ่านครบ**.
5. **ห้าม publish พร้อมข้อแม้** — "ลงก่อนเดี๋ยวแก้", "เหลือนิดเดียว", "ช่องนี้ว่างไว้ก่อน" = ยังไม่ผ่าน = ยังไม่ลง.
6. **แหล่งตัดสิน = ตารางยืนยัน Step 2 + `source-map.md` + หน้าต้นฉบับ (mock) + Wording Guideline** เท่านั้น
   ไม่ใช่ความเห็นผู้เขียน. ตารางยืนยันไม่ครอบเรื่องที่จำเป็น → **ถามผู้ใช้ อย่าเดา**.

**สัญลักษณ์ผลตรวจในทุกการ์ด:** ✅ ผ่าน (มีหลักฐาน) · ❌ ไม่ผ่าน (เจอปัญหา **หรือ** ตรวจไม่ได้) · ⚠️ ข้อสังเกต
(ไม่บล็อกเอง แต่ต้องบันทึกและอธิบาย). ⚠️ ที่อธิบายไม่ได้ = ยกระดับเป็น ❌.

## 🟦 การ์ด — ด่านที่ 1 · ตรงตามที่ผู้ใช้ยืนยัน (Requirement conformance)

**มุมมอง:** ของที่ทำ ตรงกับที่ผู้ใช้ยืนยันไว้ตอนต้น — ทีละบรรทัดของตารางยืนยัน Step 2. ไม่ขาด **และไม่เกิน**.

**โหมดตรวจ:** ด้วยตา (เทียบตารางยืนยัน).

**เกณฑ์ผ่าน — ต้อง ✅ ทุกข้อ:**
- [ ] **doc-type** ที่ทำ ตรงกับที่ยืนยัน — ไม่ไปแตะ doc-type อื่น
- [ ] **หน้าปลายทาง** ตรง — space key + cloudId + page id + title ตรงเป๊ะกับที่ยืนยัน (เขียนผิดหน้า = ผิดร้ายแรง)
- [ ] **subsystem scope** ตรง — คอลัมน์ `Subsystem` มีเฉพาะตัวที่ยืนยัน, label ติดครบและ**ไม่เกิน**
- [ ] **update in-place / สร้างหน้าลูก** ตรงที่ยืนยัน; ถ้าสร้างหน้าลูก จำนวนและรายชื่อ instance ตรงที่ยืนยัน
- [ ] **ภาษา/โทน** ตรงที่ยืนยัน (ไทย/อื่น; ทางการ; ไม่มีสรรพนาม 1/2; ไม่มีคำลงท้าย)
- [ ] **ไม่ทำเกินขอบเขต** — ไม่มีหัวข้อ/คอลัมน์/หน้า ที่ผู้ใช้ไม่ได้สั่ง

**วิธีพิสูจน์:** ทำ **ตารางเทียบ `สิ่งที่ยืนยัน → สิ่งที่ทำจริง`** ทุกแถวของตารางยืนยัน Step 2. ทุกแถวต้องมีคู่.

**หลักฐานที่ต้องแสดง:** ตารางเทียบข้างต้น (ทุกบรรทัด).

**ตกทันทีเมื่อ:** เขียนคนละหน้า/คนละ space · subsystem เกิน-ขาด · จำนวนเล่ม/หน้าลูกไม่ตรง · มีของแถมที่ไม่ได้สั่ง.

**ห้ามมโน/โกหกในด่านนี้:** ห้ามสมมติว่า page id ที่ resolve เองถูกโดยไม่ให้ผู้ใช้ยืนยัน · ถ้าตารางยืนยันไม่ได้
ระบุค่าใด **อย่าเดาว่าผู้ใช้คงหมายถึงอะไร** — ถาม.

## 🟩 การ์ด — ด่านที่ 2 · ทุกค่ามีที่มา + ไม่มี mock เหลือ (Sourcing / completeness / anti-fabrication)

**มุมมอง:** ทุกค่าที่ปรากฏสาวกลับไปถึงแหล่งจริงได้ **และ** ไม่มีของปลอม/ที่ค้างหลงเหลือ. **นี่คือด่านหัวใจของ
"ห้ามมโน"** — ตรวจละเอียดที่สุด.

**โหมดตรวจ:** ทั้งคู่ (สคริปต์ + ด้วยตาเทียบแหล่ง).

**เกณฑ์ผ่าน — ต้อง ✅ ทุกข้อ:**
- [ ] **ทุกค่าในทุกเซลล์/ทุกประโยค มีแหล่ง** — feature, ตัวเลข, ชื่อ owner, status, endpoint, column,
      วันที่, action item — ระบุได้ว่ามาจาก Jira issue ใด / บรรทัดใดของ schema / หน้าใดของ spec / โน้ตประชุมใด
- [ ] **ตรงกับแหล่งเป๊ะ ไม่แปลง** — ชื่อสะกดตรง, ตัวเลขตรง (ห้ามปัด ห้ามประมาณ), สถานะตรงคำที่แหล่งใช้
- [ ] **ไม่มี mock/placeholder token ค้าง** — `FEATnn`, `Module A/B`, `สมมติ`, `mock/MOCK`, `TBD`,
      `TODO`, `XXX`, `[ระบุ…]`, `lorem ipsum`, node `data-type="placeholder"`, panel เตือน "MOCK"
      (พิสูจน์ด้วย `verify-confluence.py`)
- [ ] **ความครบถ้วน (completeness):** ทุก placeholder ที่หน้าต้นฉบับมี → ถูกเติมด้วยค่าจริง **หรือ** ถูกลบ
      พร้อมเหตุผลที่ชัด (ไม่มีแหล่ง → ตัดทั้งหัวข้อ). **ห้ามเหลือช่องว่างเงียบๆ** — เซลล์ว่างต้องเป็น "—/ไม่มี"
      ที่ตั้งใจ ไม่ใช่ลืมเติม
- [ ] **ครบทุก subsystem ใน scope** — ทุก subsystem ที่ยืนยันมีข้อมูลจริงของตัวเอง ไม่ใช่คัดลอกค่าข้าม
      subsystem มาแปะ (ค่าเหมือนกันข้าม subsystem ต้องมีแหล่งยืนยันว่าเหมือนจริง)
- [ ] **section ที่ไม่มีแหล่ง = ตัดทิ้ง** — เช่น Troubleshooting ไม่มี known-issues → ลบ ไม่แต่งปัญหา/วิธีแก้
- [ ] **ไม่มีค่าที่ "เติมให้ดูครบ"** — ไม่มีตัวอย่างสมมติ, ไม่มีค่า default ที่คิดเอง, ไม่มีชื่อทีม/คนที่เดา

**วิธีพิสูจน์:**
1. รัน `verify-confluence.py` (ครึ่งเชิงกล: no-mock-leftover, credential, term).
2. ทำ **ตาราง traceability `ค่า/หัวข้อ → แหล่ง (Jira id / ไฟล์:บรรทัด / URL)` ครบทุกค่า**. ค่าที่เขียนแหล่งไม่ได้
   แม้ค่าเดียว → **ด่านนี้ ❌** (ไม่ใช่ ⚠️).
3. สุ่มเปิดแหล่งจริงเทียบตัวเลข/ชื่ออย่างน้อยต่อ subsystem — ยืนยันว่าตรง ไม่ใช่จำๆ มา.

**หลักฐานที่ต้องแสดง:** ผลรัน `verify-confluence.py` (ตารางเต็ม) **บวก** ตาราง traceability ทุกค่า.

**ตกทันทีเมื่อ:** เจอ mock token แม้ตัวเดียว · มีค่าที่เขียนแหล่งไม่ได้ · ตัวเลข/ชื่อไม่ตรงแหล่ง · มีเซลล์ว่าง
ที่ไม่ได้ตั้งใจ · subsystem ใน scope ขาดข้อมูล · มี section ที่ไม่มีแหล่งแต่ยังอยู่.

**ห้ามมโน/โกหกในด่านนี้:**
- ค่าที่ไม่มั่นใจ **ห้ามใส่** — หยุดถาม. "ใส่ค่าที่น่าจะใช่ไปก่อน" = มโน = ห้ามเด็ดขาด.
- ห้ามเขียนในตาราง traceability ว่า "จากแหล่ง X" ทั้งที่ไม่ได้เปิดดู X จริง = โกหก.
- สคริปต์ผ่าน **ไม่ได้แปลว่าด่านนี้ผ่าน** — สคริปต์รู้แค่ว่า "ไม่มี mock token" มันไม่รู้ว่าเลขที่ใส่ **ถูกค่า**ไหม
  นั่นคือส่วนที่คนต้องเทียบแหล่งเอง.

## 🟨 การ์ด — ด่านที่ 3 · โครง/ฟอแมตคงเดิม (Structure fidelity)

**มุมมอง:** โครงหน้าเท่าเดิมทุกอย่าง เปลี่ยนแค่ **ค่า**. ผู้ใช้ต้องการ "แทน mock ด้วยของจริง" ไม่ใช่ "ออกแบบหน้าใหม่".

**โหมดตรวจ:** ทั้งคู่ (`--original` + ด้วยตาเทียบต้นฉบับ).

**เกณฑ์ผ่าน — ต้อง ✅ ทุกข้อ:**
- [ ] **คอลัมน์ตาราง ครบ + เรียงเดิม** — ไม่เพิ่ม ไม่ลด ไม่สลับ (ยกเว้นคอลัมน์ `Subsystem` ที่เป็น convention
      ของทั้ง space — เพิ่มได้ตัวเดียวนี้)
- [ ] **heading / panel / expand / status lozenge / macro** เดิมอยู่ครบ ไม่หาย ไม่งอกใหม่
- [ ] **ลำดับ section เดิม** — ไม่สลับหัวข้อ
- [ ] **ไม่มี node `data-type="placeholder"` หลงเหลือ** และ **panel เตือน "MOCK" ถูกเอาออกแล้ว** (เป็นของ
      scaffold ไม่ใช่เนื้อหาจริง)
- [ ] **index ↔ child parity** — ทุกแถวในตาราง index ลิงก์ไปหน้าลูกที่มีอยู่จริง; ทุกหน้าลูกที่สร้างมีแถวใน index
- [ ] เขียนด้วย `contentFormat: html` (round-trip: panel/macro/table/local id ไม่หลุด)

**วิธีพิสูจน์:**
1. รัน `verify-confluence.py --original <ต้นฉบับ.html>` — เทียบ set ของคอลัมน์/มาโคร/panel/จำนวนตาราง.
2. เปิดต้นฉบับ (mock) กับ body ที่เตรียม วางเทียบกัน ทำ **ตาราง `องค์ประกอบต้นฉบับ → ยังอยู่ไหม/เปลี่ยนอะไร`**.

**หลักฐานที่ต้องแสดง:** ผล `verify-confluence.py --original` + ตารางเทียบองค์ประกอบด้วยตา.

**ตกทันทีเมื่อ:** คอลัมน์หาย/เพิ่มเกิน · panel/macro หาย · placeholder node ค้าง · warning MOCK ยังอยู่ ·
index ชี้หน้าที่ไม่มี หรือหน้าลูกไม่มีใน index.

**ห้ามมโน/โกหกในด่านนี้:** ถ้าไม่มีต้นฉบับมาเทียบ **ตรวจด่านนี้ไม่ได้ = ❌** (ต้องอ่านต้นฉบับ mock เก็บไว้ตั้งแต่
Step 3) · ห้าม "ปรับโครงให้สวยขึ้น" โดยอ้างว่าดีกว่า — ผู้ใช้สั่งคงโครง.

## 🟧 การ์ด — ด่านที่ 4 · ตัวหนังสือ ตัวเลข ศัพท์ (Text integrity)

**มุมมอง:** อ่านลื่น ไม่มีคำผิด ศัพท์ล็อกเดียวทั้งหน้า เลข/ลำดับไม่เพี้ยน โทนถูก.

**โหมดตรวจ:** ทั้งคู่ (สคริปต์ + ด้วยตาเชิงความหมาย).

**เกณฑ์ผ่าน — ต้อง ✅ ทุกข้อ:**
- [ ] **locked term เดียวทั้งหน้า** — ตรง Wording Guideline ทุกจุด ไม่มีคำพ้องปน (ผ่าน `--terms`)
- [ ] **ไม่มีคำพราก / term ถูกตัดกลาง** ด้วยช่องว่างหรือ markup (`<br>`, `</td>`, boundary) — สคริปต์จับ
- [ ] **โทน** — ไม่มีสรรพนามบุรุษ 1/2 (ผม/ฉัน/เรา/คุณ/ท่าน), ไม่มีคำลงท้าย (ครับ/ค่ะ/นะ), ไม่ใช่ภาษาแปลเครื่อง
      (Meeting notes ระบุชื่อผู้พูดได้ — ไม่ใช่สรรพนาม)
- [ ] **เลขข้อ/ลำดับต่อเนื่อง** ไม่ข้าม ไม่ซ้ำ; decimal outline ถ้ามี
- [ ] **ตัวเลขในเนื้อหาถูกค่า** (เชื่อมกับด่าน 2 — เลขต้องตรงแหล่ง ไม่ใช่แค่ "ดูเหมือนเลข")
- [ ] **ไม่มี placeholder ข้อความค้าง** — `TODO`, `[ระบุ…]`, `…`, ประโยคที่เขียนค้างครึ่ง
- [ ] **ไม่มีคำผิด/สะกดผิด** ในคำที่เครื่องตัดสินแทนไม่ได้

**วิธีพิสูจน์:** รัน `verify-confluence.py --terms "<locked terms>"` + อ่านด้วยตาหาโทน/คำผิด/ประโยคค้าง.

**หลักฐานที่ต้องแสดง:** ผล `verify-confluence.py` + จุดที่ตรวจด้วยตา (โทน/คำผิดเชิงความหมาย).

**ตกทันทีเมื่อ:** ศัพท์พ้องปน · คำพราก · เจอสรรพนาม/คำลงท้าย · เลขข้อข้าม-ซ้ำ · placeholder ข้อความค้าง.

**ห้ามมโน/โกหกในด่านนี้:** ห้ามขึ้น ✅ "ไม่มีคำผิด" ถ้ายังไม่ได้อ่านจริงทั้งหน้า · "สคริปต์ผ่าน" ไม่ครอบคลุมโทน
และคำผิดเชิงความหมาย — ต้องอ่านเอง.

## 🟥 การ์ด — ด่านที่ 5 · รูปเล่มบนหน้า Confluence จริง (Rendered page — หลัง publish เท่านั้น)

**มุมมอง:** หน้าที่ publish แล้ว **render ถูก ไม่พัง**. พิสูจน์บน **หน้าจริงเท่านั้น** — ตรงกับหลัก "รีวิวจากไฟล์ที่
export จริง" ของ manual-maker: ข้อผิดหลายอย่างเกิด**ตอนแปลง/publish** จึงเห็นได้เฉพาะบนหน้าที่ลงจริง.

**โหมดตรวจ:** ด้วยตา (อ่านกลับ + screenshot หน้าจริง). **ก่อน publish ตรวจด่านนี้ไม่ได้ = ยังไม่ต้องตัดสิน แต่จะ
ยังไม่ 5/5 จนกว่าจะ publish แล้วตรวจ.**

**เกณฑ์ผ่าน — ต้อง ✅ ทุกข้อ (หลัง publish):**
- [ ] **อ่านหน้ากลับ** (`getConfluencePage`, html) — โครงที่ตั้งใจเขียนถูกบันทึกครบ ไม่มีส่วนหาย
- [ ] **screenshot หน้า Confluence จริง** (Playwright/Chrome MCP) แล้วดูด้วยตา:
  - [ ] ตาราง render ครบ ไม่พัง คอลัมน์ `Subsystem` แสดง
  - [ ] **Mermaid/diagram macro render เป็นภาพ ไม่ใช่โค้ดดิบ ไม่ใช่กล่อง error** (ถ้า doc-type มี diagram)
  - [ ] panel/expand/status แสดงถูก
  - [ ] ลิงก์ index → child resolve ได้ (คลิกได้จริง), label ติดครบบนหน้า

CD=$(ls -d ~/.claude/plugins/cache/*/manual-maker/*/skills/confluence-docs 2>/dev/null | sort -V | tail -1); [ -n "$CD" ] || CD=~/.claude/skills/confluence-docs

    --original original-body.html \
    --terms "OLS,ELMS,CBMS,EvMS,<ศัพท์ล็อกอื่น>"

## รูปแบบรายงานผล (ต้องพิมพ์ให้ผู้ใช้เห็นทุกครั้ง — ก่อน publish แสดง 1–4, หลัง publish แสดงครบ 5)

```
รีวิว 5 ด่าน — <doc-type> @ <space>/<page id> (รอบที่ N)

| ด่าน | มุมมอง | ผล | หลักฐาน |
|---|---|---|---|
| 1 | ตรงตามที่ยืนยัน            | ✅/❌ | <ตารางเทียบ ยืนยัน→ทำจริง> |
| 2 | ทุกค่ามีที่มา + ไม่มี mock | ✅/❌ | <ตาราง traceability ค่า→แหล่ง> + <ผล verify> |
| 3 | โครง/ฟอแมตคงเดิม          | ✅/❌ | <ผล verify --original + เทียบด้วยตา> |
| 4 | ตัวหนังสือ/ตัวเลข/ศัพท์     | ✅/❌ | <ผล verify + ตรวจด้วยตา> |
| 5 | รูปเล่มบนหน้าจริง          | ✅/❌ | <อ่านกลับ + screenshot หน้า Confluence> |

สรุป: ผ่าน 5/5 → publish/ส่งได้   |   ไม่ผ่าน <n> ด่าน → แก้แล้วรีวิวใหม่ทั้ง 5 ด่าน (รอบที่ N+1)
```

- ด่าน 1–4 ต้อง ✅ ครบ **ก่อน** publish; ด่าน 5 เติมหลัง publish.
- ยังไม่ครบ 5/5 → **รายงานว่าไม่ผ่าน บอกจะแก้อะไร แล้วแก้** ห้ามบอกผู้ใช้ว่าเสร็จ ห้ามถาม "เอาไปใช้เลยไหม".
- ในตารางผล **ทุก ✅ ต้องมีข้อความหลักฐานจริงในช่องขวา** — ✅ ที่ช่องหลักฐานว่าง = ถือว่ายังไม่ได้ตรวจ = ❌.
