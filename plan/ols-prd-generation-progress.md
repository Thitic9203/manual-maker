# แผนงาน — สร้าง PRD ราย Epic ของระบบ OLS จาก Jira (confluence-docs)

> ขอบเขต: **เฉพาะ subsystem OLS (Open Learning System)** — subsystem อื่นเป็นงานคนละรอบ
>
> งานสร้างชุดเอกสาร PRD ราย Epic ในพื้นที่เอกสารของทีม โดยดึงเนื้อหาจาก Jira ทั้งหมด
> ตามหลัก **1 Story = 1 Use Case** และกฎ **ห้ามมโน · ทุกค่ามีที่มา · ตรวจไม่ได้ = ไม่ผ่าน**
>
> เอกสารฉบับนี้จงใจ **ไม่ระบุ ID ภายใน** (เลขหน้า, คีย์ตั๋ว, cloudId, ลิงก์ภายใน)
> เพราะ repo เป็นสาธารณะ — รายละเอียดเชิงพิกัดอยู่ในบันทึกงานนอก repo

## บริบทงาน

- **เป้าหมาย:** เติมโครง PRD ราย Epic ของ **subsystem OLS** ให้ครบทุก Epic — 1 หน้าต่อ 1 Epic
  ภายในแต่ละหน้าแจกแจง Use Case ทุกตัวของ Epic นั้น
- **แหล่งข้อมูลเดียว:** Jira project ของ OLS (ไม่ใช้แหล่งอื่น ไม่ลอกจากเอกสาร BA เดิม)
- **กติกาที่ยึดตลอด:**
  - 1 Story = 1 Use Case · Story ที่เป็น improvement นับเป็น UC แยก · Bug ไม่นับเป็น UC
  - ค่าที่ Story ไม่ได้ระบุ → "รอข้อมูล" ห้ามเดา ห้ามเติมให้ดูครบ
  - ข้อความขีดฆ่าใน Jira = requirement ที่ยกเลิกแล้ว → ครอบ `<s>` ไม่ยกมาเป็นกฎที่ใช้อยู่
  - ทำเฉพาะหน้าของเรา **ห้ามแตะหน้าเอกสารของ BA และห้ามแก้ Jira**

## สิ่งที่ทำเสร็จแล้ว

- ✅ สร้างหน้าย่อยราย Epic ครบทุก Epic ในขอบเขต + ตั้งชื่อรูปแบบเดียวกันทั้งชุด
- ✅ เติมตารางดัชนีหน้าแม่ (รอบแรก) พร้อม panel ระบุข้อจำกัดที่พิสูจน์ไม่ได้
- ✅ **ตรวจแหล่งข้อมูลราย UC ทั้งชุด (ด่าน 2)** — เทียบทุก UC กับตั๋ว Jira ต้นทางใหม่ทีละใบ
  - พบเนื้อจากตั๋วผิดใบปนเข้ามา **3 จุด** → แก้แล้วทั้งหมด
  - พบตั๋วขัดกันเองเพิ่ม → ใส่ panel ไม่เลือกข้าง
- ✅ **ตรวจเชิงกล (ด่าน 3–4)** ผ่าน 15/16 หน้า (เหลือ 1 หน้าตาม action ข้อ 1)
- ✅ ตัด Use Case สถานะ "To Do" ออกจาก 2 หน้า (คงหน้าไว้ + ใส่หมายเหตุระบุคีย์ + สถานะ)
- ✅ เพิ่ม panel กำกับข้อความ UC ที่สืบต้นทางในตั๋วตัวเองไม่ได้

## ตัวบล็อกใหม่ (รอบ 2026-07-21) — สิทธิ์เขียน connector

Step 0 preflight มี 2 เงื่อนไข ต้องผ่านทั้งคู่ก่อนเขียน:

1. **scope `write:page:confluence`** — รอบนี้ผู้ใช้เปิดให้แล้ว ✅
2. **tool เขียน (`updateConfluencePage` / `createConfluencePage`) โผล่ใน session** — ❌ **ยังไม่มา**

ปัญหา: Claude session ดึงรายชื่อ tool จาก MCP ตั้งแต่ตอนเปิด session — เปิดสิทธิ์/toggle ทีหลังไม่เข้า manifest
ที่กำลังรันอยู่ ค้น registry แล้วยังไม่มี tool เขียน Confluence จึง **เขียนไม่ได้** ทั้งที่ UI connector โชว์ว่าเปิด write แล้ว

**ปลดบล็อก:** reconnect MCP (`/mcp` → reconnect Atlassian) หรือปิด-เปิด Claude ใหม่ แล้วรัน `/confluence-docs` ซ้ำ
รอบใหม่ tool เขียนจะเข้า manifest แล้วเก็บงานเขียน 2 หน้าที่เหลือได้ (ข้อ 1 + ข้อ 2)

> บทเรียน: connector อ่านได้ ≠ เขียนได้ · และเปิด scope แล้วก็ยังต้อง refresh manifest (reconnect/restart) ก่อน tool ใหม่จะใช้ได้

## ความเสี่ยงที่พบระหว่างงาน (ต้องระวังในรอบต่อไป)

### connector คืนผลผิดแบบสุ่ม — พิสูจน์แล้ว ไม่ใช่การเดา
- `getConfluencePage` ด้วย id เดิม เคยคืน **หน้าคนละหน้า** และครั้งหนึ่งผิดครบ 5/5
- `searchJiraIssuesUsingJql` เคยขอตั๋วใบหนึ่ง แต่ **คืนตั๋วคนละใบ**
- `updateConfluencePage` เคยคืน **response เป็นหน้าอื่น** (แต่เขียนลงถูกหน้า — response เชื่อไม่ได้)
- **อาการผูกกับ `contentFormat`**: `html` คืนถูกราว 1 ใน 3 · `markdown`/`adf` คืนถูกทุกครั้งที่ทดสอบ

**มาตรการที่ใช้ (ต้องคงไว้ทุกครั้งที่แตะ connector):**
1. ทุกการอ่าน → เทียบ id/คีย์ที่คืนกลับกับที่ขอ **ไม่ตรง = ทิ้งทันที ดึงใหม่** (สูงสุด 5–8 ครั้ง)
2. อ่านยืนยันตัวตนด้วย `markdown`/`adf` ก่อน แล้วค่อยใช้ `html` เมื่อจำเป็นต้องได้ body
3. หลัง `update` **ห้ามเชื่อ response** ต้องอ่านกลับด้วย `markdown` ยืนยันว่าเขียนถูกหน้า
4. ครบโควตายังไม่ตรง → บันทึกว่า "ตรวจไม่ได้" **ห้ามเดา**

### Jira query บางแบบ timeout
- `parent = "<EPIC>"` และ `key in (หลายใบ)` — timeout บ่อย โดยเฉพาะ Epic ใหญ่
- `key = <ใบเดียว>` — ตอบเร็ว ใช้ได้จริง → **ดึงทีละใบเสมอ**
- มีบางตั๋วที่ timeout เฉพาะใบ แม้ query เบา (ดู action ข้อ 7)

## นิยาม "เสร็จ"

ส่งมอบได้เมื่อผ่าน **5 ด่าน 5/5**:
1. ตรงตามที่ยืนยัน — ✅
2. ทุกค่ามีที่มา — ✅ (เชิงเนื้อหา) · เหลือ 🔵 รอ BA ชี้ขาดข้อขัดแย้ง (ข้อ 5–8)
3. โครง/ฟอแมตคงเดิม — ✅ (เชิงกล) · รอปิด action ข้อ 1+3
4. ศัพท์/ตัวเลข/คำพราก — 🟡 รอปิด action ข้อ 1
5. render บนหน้าจริง — ⛔ **ตรวจไม่ได้** (action ข้อ 4) → ตามกฎ **ยังไม่ผ่าน**

> ตราบใดที่ด่าน 5 ตรวจไม่ได้ **ยังถือว่าไม่ครบ 5/5** — ไม่ประกาศว่าเสร็จสมบูรณ์
